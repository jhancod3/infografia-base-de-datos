01 · Introducción

Transformación Digital

Organización eficiente de información para decisiones inteligentes.

Visión del Proyecto

La transformación digital es esencial para las empresas modernas. Este proyecto desarrolla el modelo conceptual de una base de datos para Intrasev Motors, con el objetivo de organizar su información de inventario, ventas, clientes y empleados de manera estructurada y eficiente.

Controlar el Inventario

Gestionar motores y repuestos de alta gama con stock actualizado en tiempo real.

Registrar Ventas

Documentar cada transacción vinculando cliente, empleado y productos vendidos.

Analizar Información

Generar reportes comerciales y métricas clave para evaluar el desempeño del negocio.

Mejorar Decisiones

Transformar datos en información valiosa que guíe estrategias comerciales.

La Empresa

Especialistas en vehículos de alta gama en el corazón del Cusco.

Audi BMW Mercedes-Benz
Clientes Principales
Talleres especializados Dueños de vehículos
Problema Actual & Solución
Desorganización: Sin un sistema estructurado, el inventario de motores y repuestos es difícil de rastrear.
Ventas manuales: El registro de ventas carece de trazabilidad de empleados y clientes.
Sin análisis: Imposible generar reportes de rendimiento o stock bajo sin datos centralizados.
Solución: Un modelo relacional que conecta inventario, ventas, clientes y empleados en una estructura coherente y escalable.
03 · Requerimientos

Requisitos del Sistema

Lo que el sistema debe hacer para cubrir las necesidades de Intrasev Motors.

REQ-01
Gestión de Inventario
El sistema debe almacenar motores y repuestos con precio, categoría y stock actualizado.
REQ-02
Registro de Clientes
Guardar DNI/RUC, nombre, teléfono, correo y dirección de cada cliente o empresa.
REQ-03
Trazabilidad de Empleados
Cada venta debe registrar qué empleado la realizó para auditoría y comisiones.
REQ-04
Control de Ventas
Registrar ventas completas con todos los productos incluidos en cada transacción.
REQ-05
Reglas del Sistema
No permitir ventas con montos negativos o cero. Integridad referencial obligatoria.

05 · Relaciones

Cardinalidades del Modelo

Cómo las entidades se conectan entre sí formando el modelo relacional.

Cliente
1 → N
Venta
1 → N
DetalleVenta
N ← 1
Producto
Empleado
1 → N (Venta)
Cliente → Venta: Un cliente puede realizar muchas ventas, pero cada venta pertenece a un solo cliente.
Empleado → Venta: Un empleado puede gestionar muchas ventas, pero cada venta la atiende un solo empleado.
Venta → DetalleVenta: Una venta contiene uno o varios productos detallados en líneas individuales.
Tabla puente: DetalleVenta resuelve la relación muchos a muchos entre Venta y Producto, permitiendo que una venta tenga varios productos y un producto aparezca en varias ventas.

07 · Conclusión

Base para el Futuro Digital

Modelo Listo para Escalar
El modelo conceptual permite organizar eficientemente la información de inventario, clientes y ventas de Intrasev Motors. La estructura relacional evita redundancias, mejora el control de stock y facilita el análisis de datos comerciales.

Este diseño representa la base sólida para un futuro sistema de gestión empresarial digital, escalable a módulos de facturación, reportes avanzados y analítica de negocio.
